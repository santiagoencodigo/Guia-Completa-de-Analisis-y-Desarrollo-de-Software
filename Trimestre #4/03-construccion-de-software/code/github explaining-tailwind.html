Guía Básica de Git

Git es un sistema de control de versiones distribuido que permite rastrear cambios en archivos y coordinar el trabajo entre múltiples personas en un proyecto. Esta guía cubre los primeros pasos con Git en un repositorio local.

1. Instalación de Git

Windows

Descargar instalador desde git-scm.com

macOS

xcode-select --install brew install git

Linux (Debian/Ubuntu)

sudo apt update && sudo apt install git

Verificación

git --version

2. Creación de un Repositorio (git init)

Un repositorio Git almacena el historial de cambios de un proyecto.

  1. Crear carpeta: mkdir mi_proyecto, cd mi_proyecto.
  2. Inicializar repositorio: git init.
Nota: Esto crea una subcarpeta oculta .git que contiene los archivos necesarios para Git.

3. Adición de Archivos (git add)

Git requiere que se "preparen" los archivos antes de confirmarlos en el staging area.

# Crear archivos de ejemplo
echo "¡Hola, mundo!" > index.html
echo "body { color: blue; }" > style.css
  • Añadir archivo específico: git add index.html
  • Añadir todo: git add .
  • Verificar estado: git status

4. Confirmación de Cambios (git commit)

Un "commit" es una instantánea del proyecto. Es vital incluir un mensaje descriptivo.

git commit -m "Confirmación inicial del proyecto: se añade index.html y style.css"

La bandera -m permite añadir el mensaje directamente.

5. Navegación por el Historial (git checkout)

Git permite viajar en el tiempo a estados anteriores del proyecto.

  1. Visualizar historial: git log
  2. Navegar a un commit: git checkout [hash] (ej. a1b2c3d)
  3. Volver a la rama principal: git checkout main
